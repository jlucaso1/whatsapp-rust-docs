## Prerequisites

Before installing whatsapp-rust, ensure you have:

- **Rust 2024 edition** or later (requires recent stable Rust)
- **Cargo** package manager

<Note>
  SQLite is bundled by default with the `whatsapp-rust-sqlite-storage` crate, so you don't need to install it separately. If you prefer to link against a system-installed SQLite, disable the default `bundled-sqlite` feature.
</Note>

## Add to your project

Add whatsapp-rust and its required dependencies to your `Cargo.toml`:

<CodeGroup>

```toml Default features
[dependencies]
whatsapp-rust = "0.3"
whatsapp-rust-sqlite-storage = "0.3"
whatsapp-rust-tokio-transport = "0.3"
whatsapp-rust-ureq-http-client = "0.3"
wacore = "0.3"
waproto = "0.3"
tokio = { version = "1.48", features = ["macros", "rt-multi-thread"] }
```

```toml Minimal setup
[dependencies]
whatsapp-rust = { version = "0.3", default-features = false }
whatsapp-rust-sqlite-storage = "0.3"
whatsapp-rust-tokio-transport = "0.3"
whatsapp-rust-ureq-http-client = "0.3"
wacore = "0.3"
waproto = "0.3"
tokio = { version = "1.48", features = ["macros", "rt-multi-thread"] }
```

</CodeGroup>

## Feature flags

whatsapp-rust supports several optional features:

| Feature | Description | Included by default |
|---------|-------------|--------------------|
| `simd` | SIMD-optimized binary protocol encoding/decoding | ✅ Yes |
| `sqlite-storage` | SQLite storage backend | ✅ Yes |
| `tokio-transport` | Tokio WebSocket transport | ✅ Yes |
| `ureq-client` | Ureq HTTP client | ✅ Yes |
| `tokio-native` | Tokio multi-threaded runtime | ✅ Yes |
| `signal` | Unix signal handling | ❌ No |
| `danger-skip-tls-verify` | Skip TLS verification (unsafe) | ❌ No |
| `debug-snapshots` | Debug protocol snapshots | ❌ No |
| `debug-diagnostics` | Memory diagnostics API (`MemoryDiagnostics`) | ❌ No |

The `whatsapp-rust-sqlite-storage` crate has its own feature flags:

| Feature | Description | Included by default |
|---------|-------------|--------------------|
| `bundled-sqlite` | Bundles SQLite (compiles from source, no system library needed) | ✅ Yes |

To use a system-installed SQLite instead of the bundled version:

```toml Cargo.toml
[dependencies]
whatsapp-rust-sqlite-storage = { version = "0.3", default-features = false }
```

<Note>
  The default features provide everything needed for most use cases. Only customize features if you have specific requirements.
</Note>

### wacore-binary features

The `wacore-binary` crate has its own feature flags:

| Feature | Description | Included by default |
|---------|-------------|--------------------|
| `simd` | SIMD-accelerated encoding/decoding (requires nightly Rust) | ✅ Yes |
| `serde` | Serde serialization support | ❌ No |

<Tip>
  To use `wacore-binary` on **stable Rust**, disable the `simd` feature: `cargo build -p wacore-binary --no-default-features`. A scalar fallback provides equivalent functionality.
</Tip>

## Custom features example

If you want to use only specific features:

```toml Cargo.toml
[dependencies]
whatsapp-rust = { version = "0.3", default-features = false, features = ["sqlite-storage", "tokio-transport"] }
```

## Verify installation

Create a simple test file to verify the installation:

```rust src/main.rs
use whatsapp_rust::bot::Bot;

fn main() {
    println!("whatsapp-rust installed successfully!");
}
```

Run it with:

```bash
cargo run
```

If you see "whatsapp-rust installed successfully!", you're ready to move on to the [Quickstart](/quickstart) guide.
